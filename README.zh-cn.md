# 模板替换

## 安装

- node:
  `pnpm add gooplate`
- deno:
  `deno add jsr:@fdcn/gooplate`

## 用法与例子

```javascript
import {convert} from "gooplate";
let tags = {
    "name": "GD8",
    "ID": 8078,
    "peoples": [{
        "name": '张三',
        "age": 15,
    }, {
        "name": '李四',
        "age": 22,
    }, {
        "name": '王五',
        "age": 19,
    }, {
        "name": '赵六',
        "age": 29,
    }]
};
let template = `group: {{name}}{{ID = ID-8000 }}{{// 说明：取ID后2位形成编号}}
ID: {{ID}}
people:{{for people in peoples}}_
{{  if people.age > 18}}
- {{people.name}}_
{{  endif}}_
{{endfor}}
`
console.log(convert(tags, template));
```

则输出：

```plaintext
group: GD8
ID: 78
people:
- 李四
- 王五
- 赵六
```

如果要应用多条转换规则，用`convert_rules`函数，见  [`./example/example.entry.js`](./example/example.entry.js)。

## 概念

gooplate提供在文本中将替换符转换成对应模板内容，效果上类似于 react vue 的数据替换，但只用于文本。它没有格式，无法获知所在文本是 html 还是 markdown 等。

主要有以下五类模板替换：

### 表达式替换

计算表达式，并在所在位置替换成结果。

可以在表达式中使用JS语法的变量、字面量、运算符、方法。注意，字面量和运算符有限制，见附录。

`{{表达式}}`

其中，`{{`称为首替换符，`}}`称为尾替换符。

在表达式中还可以使用转换器的内置函数:

- `range(start, end, step)` 产生一个递增或递减的生成器对象，从 start 开始，小于end，步长为 step。
- `stepper(start, step)` 产生一个递增或递减的步进器，s.next() 步进到下一个值并返回, s.value 为当前值。

非整数时，请注意输出为浮点数，建议输出时用 toFixed(n) 进行格式化。

### 赋值

建立一个新变量，并赋值表达式的值。赋值不会产生替换，可以在后继替换中使用新建变量。
`{{varname = 表达式}}`

### 条件替换

根据条件决定是否显示对应结果，`elseif` `else` 是可选的，`elseif` 可以有多个。
1. `{{if 条件1}} 输出1 {{endif}}`
2. `{{if 条件1}} 输出1 {{else}}其它输出 {{endif}}`
3. `{{if 条件1}} 输出1 {{elseif 条件2}} 输出2 {{else}} 其它输出 {{endif}}` 

### 循环替换

- `{{for value in object}} 内容:{{value}} {{endfor}}`
    循环输出，遍历object的所有自身属性值（不是属性键值）。
- `{{for key, value in Object}} 键值:{{key}}, 属性值:{{value}} {{endfor}}`
    同时遍历键和值，当object为Array时，键值为数字。

### 注释

在任何模板语法中，都可以用 `// 注释文本` 用于注释，不产生输出，`//` 与该模板尾替换符之间的任何文字都视为注释。

## 扩展替换符

除了常规的首替换符`{{`和尾替换符`}}`外，还有一些可以影响到替换符外部空白文字的扩展替换符。

使用扩展替换符，主要是为了在视觉上方便查看替换结构。可以参看 `example/example.js` 示例。

以下示例中，假设 code 变量的值为 `@`

### 左侧空白移除替换符

为了方便书写模板，可以使用 `{{_` 消除替换符左侧的空白，但不包括换行。这些空白不会输出到正文中。

- `A 	  {{_code}}` 输出→ `A@`。

### 左侧合并行替换符

如果想把空白行也消除掉，使用变种 `_{{` ，它实现将上方第一个非空白行与当前行 `{{` 之后的内容连成一行。 

具体实现过程为:

- 它回溯到上方最近一个非空白行，消除该行末尾的换行符，以及该换行符之后、`_{{` 之前的所有空白。非空白行自身的尾部空白保留。
- 当 `_{{` 左侧不存在换行符，或左侧第一个换行符与 `_{{` 之间有非空白字符时，_ 回退为普通字符输出，而非作为空白消除标记。

- `A 
 	   _{{code}}` 输出→  `A @`
- `A  	   _{{code}}` 输出→  `A  	   _@`

注，`_{{_code}}` 两侧都有 `_` 时，只有 `{{_` 起作用。具体见 §扩展替换符歧义与转义

### 右侧空白移除替换符

为了方便书写模板，可以使用 `_}}` 消除替换符右侧的所有空白，不会输出到正文中。

- `{{code_}}  	 A` 输出→ `@A`

### 右侧合并行替换符

如果想把空白行也消除掉，使用变种 `}}_`，它实现将当前行 `}}` 之前的内容与下方第一个非空白行连成一行。 

- 它消除从 `}}_` 之后到下一个非空白行之前的所有空白和换行。非空白行自身的首部空白保留。
- 当 `}}_` 右侧不存在换行符，或 `}}_` 与右侧第一个换行符之间有非空白字符时，_ 回退为普通字符输出，而非作为空白消除标记。

- `{{code}}_
 	   
 A` 输出→  `@ A`
- `{{code}}_  	   A` 输出→ `@_  	   A`

注，`{{code_}}_` 两侧都有 `_` 时，只有 `_}}` 起作用。具体见 §扩展替换符歧义与转义

### 扩展替换实例

```
1{{// 删除后导空白 _}}    这一行会删除空白
2{{ }}_        	
    这一行前的空白会尾随在上一行末输出正文中，即不保留上一行末的空白与换行
3{{ }}_


    这一行会尾随在3后输出正文中，上3行空白行的换行符会被移除

    		_{{//会在上个非空白行末尾加个句号}}。
4    A_{{//这2个_符号不会被识别为扩展替换符}}_B
        {{_ //删除前导空白}}完结
```

最终的输出为：

```
1这一行会删除空白
2    这一行前的空白会尾随在上一行末输出正文中，即不保留上一行末的空白与换行
3    这一行会尾随在3后输出正文中，上3行空白行的换行符会被移除。
4    A__B
完结
```

### 扩展替换符歧义与转义

- 当尾替换符后必须输出一个 `_<blank_line>` 时，可以使用 `_}}_` ，`_}}` 提前中止了尾替换符，随后的 `_` 正常输出。
- 当首替换符前必须输出一个 `<newline_with_blank>_` 时，可以使用 `_{{_` ，`{{_` 会前瞻阻止 `_{{` 的识别，前面的 `_` 正常输出。
- 当替换符内部有 `_` 做前后缀的变量时，不要与替换符相连，以防识别为扩展替换符。比如 `{{ _var1 +  var2_ }}`
- `{{_}}` 是非法的，程序无法判断扩展首替换符还是扩展尾替换符，转换会中止。可以通过模板内加空格进行正确标记，如 `{{_ }}` 或 `{{ _}}`。

## 附录

**字面量**

目前支持布尔值、数值、字符串、数组和对象。

**受支持的运算符**

| 类别 | 操作符 |
|---|---|
| 一元运算符 | + - ~ ! |
| 二元运算符 | in + - * / % == === != !== < > <= >= |
| 三元运算符 | ?: |
| 赋值运算符 | = += -= *= **= /= %= ??= |
| 成员运算符 | obj.prop obj[prop] |
| 序列运算符 | , |
| 逻辑运算符 | \|\| && ?? |
| 扩展运算符 | ... |
| 调用运算符 | () |
